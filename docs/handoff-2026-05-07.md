### 2.1 `e3b4646` — 블로그 이미지 보존 (issues 기반 패치)

**파일**: `next-app/app/(dashboard)/blog/page.tsx` (+109/−39)

**문제**: Opus 4.7 검수가 돌려준 `revisedHtml`에서 `<img>` 태그가 비결정적으로 누락됨. 기존 방어 로직이 이미지 손실 감지 시 원본을 유지 → 의료법 보정이 통째로 폐기되는 트레이드오프.

**변경**: `revisedHtml` 통째 적용을 폐기. 대신 `review.issues[].originalQuote → suggestion`을 원본 `blogText`(이미지 박힌 상태)에 직접 substring 치환. HTML 구조 미접촉 → 이미지 손실 구조적으로 불가능. 매치 실패는 `applyContentFilters` 정규식 안전망에 위임.

**텔레메트리** (모니터링 핵심):
```
[BLOG] [V4] issues 패치 적용 — applied=N, skipped=M, img=A→B, revisedHtml=lost_imgs|ok|absent
```
- `revisedHtml=lost_imgs` 분포: 기존 방식이었으면 이미지를 잃었을 케이스 (이번 패치가 실제로 구해낸 수)
- `skipped > applied`가 잦으면 originalQuote 매치율이 낮은 것 → 3차 전략(HTML 태그 strip 후 매치) 추가 검토

**롤백**: revert commit `e3b4646`. 헬퍼 `applyIssuesPatch`는 `BlogForm()` 위 모듈 레벨에 추가됨.

**미반영**: `public-app`도 동일 버그 보유. 검증 후 동일 패치 포팅 필요.

### 2.2 `34363ac` — PSI 타임아웃 정합 (25s → 70s)

**파일**: `next-app/app/api/diagnostic/route.ts` (+4/−1)

**문제**: AEO/GEO 진단의 PSI(PageSpeed Insights) 호출이 느린 사이트(Lighthouse Mobile 25-40초)에서 항상 실패. `route.ts`가 `withTimeout(fetchPsi, 25_000)`로 감싸는데 내부 `psi.ts`는 35초×2회 시도 = 최악 70초 budget로 설계됨. 외부 wrapper가 먼저 abort → 재시도 코드는 dead retry.

**변경**: wrapper budget을 `25_000` → `70_000`으로 변경. `psi.ts` 코멘트의 설계 의도와 정합.

**예산 검증**: crawl 40s + psi 70s + enrich 40s = 150s. Vercel `maxDuration=180` 안에 정합.

**롤백**: revert commit `34363ac`.

**부수 개선 후보** (이번엔 미적용):
- `withTimeout`에 `AbortController` 전파해 wrapper abort 시 inner fetch도 cancel — 현재는 백그라운드에서 inner promise가 계속 돌아 리소스 낭비
- PSI를 crawl과 부분 병렬화

### 2.4 `10c11c5` — 행동 기반 장소 분기

**파일**: `next-app/app/api/image/route.ts` (+13/−8)

**문제**: 위 2.3 룰 적용 후, 양치·치실·치간칫솔 같은 가정용 행동까지 치과 의자에서 그려지는 부작용 발견. `[KOREAN MEDICAL CONTEXT]` 블록이 무조건 "clinic interior"를 강제.

**변경**: 절대적 medical-interior 블록을 **action-driven `[LOCATION]` 테이블**로 교체:

| 행동 | 장소 |
|---|---|
| 진료/시술/상담/X-ray | clinic operatory / consultation room |
| 양치·치실·치간칫솔·구강세정·스킨케어·복용 | 한국 가정 욕실 (세면대+거울) — clinic 명시적 금지 |
| 식사·회복식·수분섭취 | 한국 가정 다이닝/주방 |
| 운동·재활 | 공원·거실·홈짐 — 진료의자 금지 |
| 일상 (산책·독서·대화) | 카페·공원·거리·집 |

`[CULTURAL ANCHORING]`을 별도 블록으로 분리해 한국인 피사체·따뜻한 분위기는 보편 유지.

**롤백**: revert commit `10c11c5`. 단, 2.3과 함께 롤백하면 분할 이미지 회귀 위험.

**미반영**: `public-app`은 본 변경 미적용. 행동 기반 장소 분기는 양쪽 동일 적용이 자연스러움.

### 2.5 `b2401af` — HARD OVERRIDE 블록 추가 (upstream clinic-bias 무력화)

**파일**: `next-app/app/api/image/route.ts` (+13)

**문제**: 위 2.4 적용 후에도 "여성이 진료의자에 앉아 죽 먹는 장면"이 또 발생. `body.prompt`는 `blog-core/buildImagePrompt`에서 `categoryHints[category]` (= `'dental clinic setting, modern minimalist Korean dental office'`)을 prefix로 baked in해서 옴. gpt-image-2가 더 구체·후순위 cue를 우선시 → 내가 위에 둔 LOCATION 룰을 override.

**변경**: `body.prompt` **뒤에** `[HARD OVERRIDE]` 블록을 추가. 모델이 마지막에 읽는 controlling instruction이 되어 우선순위가 가장 높음:
- 식사·음용·회복식 → 한국 가정 다이닝/주방. clinic / 진료의자 / 의료 도구 일체 금지
- 양치·치실·치간칫솔·구강세정·스킨케어 → 한국 가정 욕실 또는 따뜻한 가정 인테리어. clinic 의자 일체 금지
- 실제 임상 시술만 clinic 사용
- "이전 줄에 'clinic setting' 언급이 있어도 행동이 비임상이면 그 줄을 무시하라"

**롤백**: revert commit `b2401af`. 단, 2.3·2.4와 함께 롤백하면 분할 이미지·clinic-bias 모두 회귀.

**더 깨끗한 구조적 수정 후보** (이번엔 미적용 — 항목 5의 결정 대기 사항):
- `blog-core/buildImagePrompt`의 `categoryHints` 자체에서 "dental clinic setting" 같은 location-forcing 어휘를 제거하고, 토픽 anchoring만 유지. 양쪽 앱(next-app, public-app)에 동일하게 영향. 이 결정은 시니어가 내리는 게 맞음
